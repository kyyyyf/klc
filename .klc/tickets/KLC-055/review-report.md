### V2 ‚Äî fresh LOW-2 ‚Äî new `SystemExit` behaviour (awareness note, no change)

`identity.current()` raises `SystemExit` when no git identity and no `$USER`
is configured, replacing the old `_git_user()` `"unknown"` fallback. The fresh
reviewer confirmed this is coherent, not a latent crash: `intake` is a
user-initiated CLI entrypoint (`if __name__ == "__main__": sys.exit(run(...))`)
with no programmatic `intake.run(` callers outside tests, so `SystemExit`
prints the setup instruction to stderr and exits non-zero ‚Äî the intended
fail-fast. The only residual note: a bare-container e2e run with git identity
unset AND `$USER` empty would now hard-exit; `$USER` is normally set.

**Assessment: no change.** Intentional behaviour change, correct for a CLI
entrypoint (satisfies AC-2). Documented here for awareness.
